Aurora (Heavy Data) 
- 결제 내역, 주문내역 등의 주요한 데이터, 로그 쌓임
- Aurora는 주문/결제와 같이 트랜잭션과 데이터 정합성이 중요한 데이터를 저장
ACID를 보장하며 복잡한 관계형 쿼리에 적합
DynamoDB
- 장바구니, 로그인 상태 유지 사용자 세션 등 단순한 구조이지만 요청이 수시로 들어오는 데이터, 서버가 오토스케일링되도 성능이 일정하게 유지
DynamoDB는 서버리스 AWS가 알아서 처리량을 늘려줌, 갑작스러운 이벤트 트래픽을 받아내기에 최적.
- VPC 엔드포인트,DynamoDB Endpoint 덕분에 트래픽이 비싼 인터넷 망을 거치지 않고 내부망으로만 왔다 갔다함. 비용 절감과 보안의 측면 강화
- SQS는 비동기 방식으로써 특정 서비스에 장애가 나도 전체 시스템은 멈추지 않음.
트래픽 폭증 시 서버가 느려질 순 있어도, 과부하로 서버가 나가는 상황 방지 가능
- 결제 데이터는 서버가 죽고 다시 살아나도 SQS에 남아있어서 처리 못 한 메시지를 다시 가져와서 작업 마무리 가능
- 서비스 간 결합도를 낮추고 웹 서버와 결제 처리를 분리하여 트래픽 급증 시에도 안정적으로 처리
